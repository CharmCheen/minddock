# SRS 需求规格说明书（SRS）
**项目名称**：基于 AI 智能平台的个人知识管理助手（Agentic RAG PKM Assistant）  
**版本**：v1.0  
**日期**：2026-03-04  
**文档状态**：初版（可用于开题/中期/答辩材料支撑）

---

## 0. 摘要
本系统面向个人多源知识资产（本地文档、网页剪藏、笔记导出、日程/待办等），构建可持续更新的“外置长期语义记忆（向量索引 + 元数据）”，并通过检索增强生成（RAG）与工作流技能编排（Skills/Workflow）实现跨文档问答、主题总结、结构化输出（如 Mermaid 思维导图）与可选的主动提醒。系统强调**证据驱动（答案必须附引用）**、**可配置（通用场景可切换）**、**可评测（检索与引用一致性指标）**。

---

## 1. 术语与缩略语
- **PKM**：Personal Knowledge Management，个人知识管理  
- **LLM**：Large Language Model，大语言模型  
- **RAG**：Retrieval-Augmented Generation，检索增强生成  
- **Chunk**：文档分块片段  
- **Embedding**：向量化表示  
- **Vector DB**：向量数据库/向量索引  
- **Workflow/Skills**：工作流/技能链路（可复用任务流程）  
- **Citation**：引用/出处（证据片段与来源定位）

## 2. 背景、目标与范围

### 2.1 背景与痛点（Problem Statement）
1) 个人知识分散于多应用与多格式，形成“知识孤岛”；  
2) 传统检索偏关键词，难以跨文档推理与归纳；  
3) 现成大模型受上下文窗口限制，无法一次性加载长期积累的大量文档；  
4) 裸生成缺乏可追溯证据链，存在幻觉风险；  
5) 知识复用率低，知识库易沦为“数字垃圾场”。

### 2.2 项目目标（Project Objectives）
- **O1**：实现多源知识接入与规范化入库（解析/清洗/分块/向量化/索引）。  
- **O2**：实现证据驱动的跨文档问答（答案 + 引用证据）。  
- **O3**：实现多文档主题总结（Map-Reduce），并输出可追溯引用。  
- **O4（推荐）**：实现结构化输出技能（Mermaid 思维导图/提纲 JSON）。  
- **O5（可选）**：实现主动提醒/简报（定时触发 + 推送）。  
- **O6**：提供“通用型场景配置”能力：通过 Prompt 预设/配置文件切换输出风格、检索范围与技能组合。

### 2.3 项目范围（In Scope）
- 单用户个人知识库（不做多用户协作与复杂权限体系）。  
- 数据源至少覆盖：**本地文件 + 网页剪藏**（或笔记导出）。  
- 以 Coze（扣子）等可视化 AI 平台作为 Agent 编排与交互主入口（可替换实现需在论文中说明）。  
- 支持最少 2 个核心技能：跨文档问答、主题总结。

### 2.4 非范围（Out of Scope）
- 训练/微调专属大模型（可作为讨论但不作为必须实现）。  
- 企业级权限/审计/多租户。  
- 全自动知识图谱抽取与复杂图推理（可作为扩展模块讨论）。  
- 强实时同步与强一致性（采用异步增量更新为主）。

## 3. 典型用户画像与使用场景

### 3.1 用户画像
- 高校学生/科研人员：文献、课堂资料、学习笔记、网页剪藏。  
- 研发/产品/运营：需求文档、会议纪要、竞品资料、周报。  
- 通用知识工作者：阅读摘要、知识复盘、写作辅助。

### 3.2 主场景（通用可配置）
系统提供“**场景配置 Profile**”，用户可在对话中或配置文件中切换：
- **学习/论文**：强调严谨引用、术语一致、对比表与复习导图；  
- **项目/会议**：强调行动项、时间线、决策依据与责任人；  
- **写作/报告**：强调结构化大纲、论据归纳与可复用段落。

> 核心原则：同一套引擎（RAG + Workflow）通过 **Prompt 预设 + 检索过滤策略 + 输出模板** 实现“可微调通用型”。

## 4. 总体需求概览

### 4.1 功能性需求（Functional Requirements, FR）
**FR-1 多源接入与入库**
- FR-1.1 支持上传/导入本地文件（至少 PDF 与 Markdown/Docx 两类之一）。  
- FR-1.2 支持 URL 输入的网页正文抽取并入库（网页剪藏）。  
- FR-1.3 支持文档元数据（来源、时间、标签、类型）录入/自动生成。  
- FR-1.4 支持增量更新（同一文档再次导入时 Upsert 更新）。

**FR-2 文档解析与清洗**
- FR-2.1 能从文本型 PDF 抽取正文与页码定位（至少记录页码范围）。  
- FR-2.2 清洗噪声（多余空行、脚注噪声、导航栏等）。  
- FR-2.3 支持可配置分块策略（按标题/段落优先，超长再递归切分）与 overlap。

**FR-3 向量化与检索（RAG 基础）**
- FR-3.1 对分块内容进行 embedding 并写入向量索引（平台知识库或自建向量库）。  
- FR-3.2 支持检索 top-k，并输出 chunk 列表与相似度/排序信息。  
- FR-3.3 支持元数据过滤（时间范围、来源、标签）。  
- FR-3.4（可选）支持 rerank（规则或模型）提升精度。

**FR-4 跨文档问答（必做）**
- FR-4.1 输入问题与检索范围，输出结构化答案。  
- FR-4.2 输出必须包含引用：每个主要结论至少 1 条证据引用（doc_id/标题 + chunk_id + 定位信息）。  
- FR-4.3 当证据不足时输出“证据不足/无法支持结论”并给出建议检索方向。

**FR-5 多文档主题总结（必做）**
- FR-5.1 支持多文档 Map-Reduce：先分文档局部总结，再全局融合。  
- FR-5.2 输出包含：关键要点、对比/趋势、行动项（可配置），并附引用。

**FR-6 结构化输出（推荐）**
- FR-6.1 支持导出 Mermaid mindmap 或结构化 JSON 大纲。  
- FR-6.2 输出可直接渲染或被前端/平台渲染。

**FR-7 个性化配置与偏好记忆（必做轻量版）**
- FR-7.1 支持设置输出模板（如“结论-证据-行动”）、语言偏好、引用样式。  
- FR-7.2 偏好可持久化并在后续对话/技能中生效。  
- FR-7.3 支持 Profile 切换（学习/项目/写作）。

**FR-8 主动提醒/简报（可选）**
- FR-8.1 支持定时触发（每日/每周）执行简报工作流。  
- FR-8.2 支持读取日程/待办或用户指定任务列表并生成提醒。

---

### 4.2 非功能性需求（Non-Functional Requirements, NFR）
- **NFR-1 可用性**：核心技能（问答、总结）响应时间可接受（建议 < 10–20s，视模型与平台而定）。  
- **NFR-2 可追溯性**：所有知识型输出须可回溯到具体文档片段。  
- **NFR-3 鲁棒性**：解析失败或检索为空时应返回明确错误信息与降级策略。  
- **NFR-4 可维护性**：技能以工作流形式组件化，可独立迭代；工具接口有版本号。  
- **NFR-5 可扩展性**：可新增数据源、技能、检索策略而不破坏既有接口。  
- **NFR-6 隐私与安全**：个人数据只在授权范围内处理；敏感数据不写入公开日志；提供数据清理/删除入口（至少文档级删除）。  
- **NFR-7 可评测性**：支持导出日志与评测数据（命中率、引用一致性、人工评分表）。

## 5. 业务流程与用例（Use Cases）

### 5.1 用例 UC-1：导入文档并入库
- **参与者**：用户  
- **前置条件**：平台可用；用户提供文件或 URL  
- **主流程**：上传/输入 → 解析 → 清洗 → 分块 → 向量化 → 入库 → 返回文档卡片  
- **后置条件**：文档可被检索与引用  
- **异常**：解析失败→提示原因与可重试策略（换解析器/手动粘贴文本）

### 5.2 用例 UC-2：跨文档问答（带引用）
- **参与者**：用户、Agent  
- **主流程**：问题输入 →（可选）query rewrite → 检索 top-k →（可选）rerank → 组装上下文 → 生成答案 → 输出引用列表  
- **异常**：检索为空→提示“未命中”并建议补充关键词/放宽过滤

### 5.3 用例 UC-3：主题总结（Map-Reduce）
- **主流程**：主题/范围输入 → 检索候选文档/片段 → 分文档局部总结（Map）→ 全局融合（Reduce）→ 输出报告 + 引用  
- **异常**：候选过多→自动采样或分批 Map，保证上下文不溢出

### 5.4 用例 UC-4：思维导图生成（推荐）
- **主流程**：主题输入 → 检索 → 生成层级结构 → 转 Mermaid → 输出可渲染结果

### 5.5 用例 UC-5：Profile 切换与偏好设置
- **主流程**：用户设置（例如“以后输出按：结论-证据-行动”）→ 写入偏好 → 后续技能自动应用  
- **可选**：提供预设 Profile（学习/项目/写作）一键切换

## 6. 数据需求与数据模型（Data Requirements）

### 6.1 核心数据对象（建议）
**Document**
- doc_id, title, source_type (local/url/notion/...), source_uri, created_at, updated_at  
- tags[], time_range, checksum/hash, parse_status

**Chunk**
- chunk_id, doc_id, text, token_count, section_path（标题层级）, location（页码/段落锚点）  
- embedding_vector（或向量库内部 id）, score（检索时）

**Preference/Profile**
- user_id（单用户可省略）, profile_name  
- output_template, language, citation_style, default_filters, enabled_skills

### 6.2 引用格式（必须固化）
- 引用条目包含：文档标题/来源 + chunk_id + location（页码/锚点）+ 引用片段（可截断）。  
- 支持脚注式或列表式引用（由偏好决定）。

## 7. 外部接口与工具（Tools/Plugins）规格

### 7.1 Tool-1：Web Extractor
- **Input**：{"url": "...", "mode": "readability|full", "timeout_s": 10}  
- **Output**：{"title": "...", "markdown": "...", "text": "...", "meta": {...}}  
- **Failure**：{"error_code": "...", "message": "..."}

### 7.2 Tool-2：Ingest Document
- **Input**：{"content": "...", "title": "...", "source_uri": "...", "tags": [...], "time": "YYYY-MM-DD"}  
- **Output**：{"doc_id": "...", "chunk_count": 123, "status": "ok"}  

### 7.3 Tool-3：Vector Search
- **Input**：{"query": "...", "top_k": 8, "filters": {"tags":[], "time_from":"", "time_to":"", "source_type":[]}}  
- **Output**：{"hits":[{"doc_id":"", "chunk_id":"", "text":"", "score":0.0, "location":""}]}  

### 7.4 Tool-4：Citation Builder
- **Input**：{"hits":[...], "style":"footnote|list"}  
- **Output**：{"citations":[{"id":"C1","ref":"...","quote":"...","location":"..."}]}  

### 7.5 Tool-5（可选）：Schedule Reader / Notifier
- 读取日程/待办并推送消息（具体依赖平台能力与授权方式）

## 8. 场景配置（Profile）机制（通用型可微调核心）

### 8.1 Profile 配置示例（YAML/JSON 思路）
```yaml
profile_name: "study"
enabled_skills: ["ask", "summarize", "mindmap"]
default_filters:
  tags: ["论文", "课程"]
  time_range_days: 180
output:
  language: "zh"
  format_template: |
    ## 结论
    - ...
    ## 证据（引用）
    {citations}
    ## 下一步行动
    - ...
citation_style: "footnote"
retrieval:
  top_k: 8
  rerank: false
safety:
  require_citations: true
  refuse_without_evidence: true
```

### 8.2 Profile 生效点
- Prompt 系统提示词/模板  
- 检索过滤参数（tag/time/source）  
- 输出格式与引用样式  
- 技能启用/禁用（可退可进）

## 9. 验收标准（Acceptance Criteria）

### 9.1 功能验收（建议最低）
- AC-1：可导入 ≥ 20 篇文档/剪藏并完成入库。  
- AC-2：跨文档问答支持引用输出：每次回答至少 3 条引用证据（或按结论条数对应）。  
- AC-3：多文档总结支持 Map-Reduce，并给出引用。  
- AC-4：偏好设置后输出格式稳定遵循（连续 3 次任务一致）。  
- AC-5：错误/空召回时有降级提示与建议。

### 9.2 质量验收（论文评测）
- QAC-1：Top-k 命中率（人工标注：问题相关证据是否在 top-k 内）。  
- QAC-2：引用一致性（答案中的关键句是否能在引用片段中找到支持）。  
- QAC-3：人工总体评分（正确性/可用性/可读性，1–5 分量表）。  
- QAC-4（可选）：对比实验（不同 chunk size/top-k/是否 rerank）。

## 10. 风险与约束
- **解析质量风险**：复杂 PDF 版式导致脏文本；需提供增强解析或降级方案。  
- **检索质量风险**：分块与 top_k 参数敏感；需基于评测数据调参。  
- **平台约束**：部分能力受 Coze/模型配额/触发器限制；需在论文中说明。  
- **隐私与合规**：个人文档与 API 授权需谨慎处理，避免泄露与无授权访问。

## 11. 版本迭代建议（可进可退）
- **MVP**：本地文件 + URL；技能=问答+总结；偏好=输出模板；平台知识库。  
- **标准版**：+导图；+更完善引用卡片；+评测报告。  
- **进阶版**：+自建向量库/混合检索/rerank；+检索评估器（二次检索）；+主动提醒。

## 12. 附录：最小演示脚本（Demo Script）
1) 上传 1 份 PDF + 粘贴 1 个 URL → 展示入库状态与文档卡片  
2) 提问跨文档问题 → 输出答案 + 引用证据  
3) 执行主题总结 → Map-Reduce → 输出报告 + 引用  
4)（可选）生成思维导图 → Mermaid 输出并渲染  
5) 设置偏好（输出模板/引用样式）→ 再问一次 → 验证偏好生效
